This page demonstrates common Angular decorators: @Component, @Input, @Output.

Simulated Parent Area

In real projects, the parent component provides data and listens to events. Here, we show the concepts without touching app.html.

Likes: {{ likesCount }}

When you click, the component method runs, updates state, and Angular updates the DOM. (If a parent were listening, it would also receive an @Output event.)

Quick Summary

  • @Component: tells Angular this class controls a piece of UI.
  • @Input(): parent passes data into the child (like function parameters).
  • @Output(): child sends an event back to parent (like callbacks).
